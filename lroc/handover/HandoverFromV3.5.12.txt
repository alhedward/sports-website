SUMMARY OF THIS PASS
====================
Tony reported that in Trips & Events, clicking the map lookup action showed:

  Could not find that location: Geoapify geocoding API key is not configured.

Changes made:

- The admin geocode route now returns a clean configured=false response when the Geoapify geocoding key is missing, instead of bubbling this up as a location lookup failure.

- The Trips & Events admin UI now treats missing geocoding config as a configuration/availability issue, not as “could not find that location”.

- The Open/Search in maps link in the admin event editor now works from meeting name/address even before lat/lng are known. Previously it only appeared once coordinates existed.

- Terraform now falls back to using the Geoapify map tile key for geocoding if a separate geocoding key is not supplied. This matches Geoapify’s usual account API key model and avoids needing two separate variables for normal LROC deployments.

- Added GEOAPIFY_GEOCODING_URL to the member-files Lambda environment so the existing Terraform variable is actually honoured.
