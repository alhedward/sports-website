Problem fixed in v3.4.4
-----------------------
Tony tested v3.4.3 and found that the new TinyMCE article renderer was now
placing article pages sequentially, but generated pages still had several
editing/rendering problems:

- The external Article Library title/header appeared at the top of every rendered
  article page. This should not happen; the TinyMCE/article body owns real page
  headings.
- Setting the optional page header/headline override to a single space could alter
  behaviour. After doing this, page 1 could appear in both the first and second
  selected/generated pages, with the final generated page appearing on the third.
- Before changing that optional page header field, the first page could leave a
  noticeable empty area near the bottom-right of the rendered page.

Root causes
-----------
1. The normal Issue Builder content-slot wrapper still treated `mce_article_page`
   like a generic content slot, so it rendered the Article Library title above the
   generated article page.

2. When a generated `mce_article_page` was opened in the page editor and saved,
   the dynamic slot-definition/readback path did not preserve the generated
   `articlePageHtml` fragment or all of the TinyMCE page metadata. That meant the
   preview could fall back to the full article HTML and show the start of the
   article again on a continuation page.

3. Whitespace-only overrides were saved as truthy text, so a single space could
   act like an intentional page/header override.

What v3.4.4 changed
-------------------
Primary front-end files changed:

  lroc/site/app.js
  lroc/site/styles.css
  lroc/site/service-worker.js

Version files changed:

  lroc/VERSION
  lroc/changelog/changelog_3.4.4.md
  lroc/handover/HandoverFromV3.4.4.txt

Implementation details:

- Added `mce_article_page` to the render modes that suppress wrapper slot titles.
- Trimmed whitespace-only headline/page-header overrides to blank when reading
  slot editor values.
- Preserved generated TinyMCE page fields through dynamic slot definition and
  editor save/read paths:
  - `articleHtml`
  - `articlePageHtml`
  - `mcePageIndex`
  - `mcePageNumber`
  - `mcePageCount`
  - `mcePageOffsetPx`
  - `mcePageHeightPx`
  - `mcePageContentHeightPx`
  - `mcePageFooterHeightPx`
  - `mcePageWidthPx`
  - `mcePaginationMode`
  - `mcePaginationDiagnostics`
  - `mceMeasuredHeightPx`
  - `documentColumns`
  - `sourceLayoutMode`
- Added preview recovery for older/misaligned generated pages: if `articlePageHtml`
  is missing but the full article HTML and page index/count are present, the
  preview attempts to repaginate and select the correct page fragment.
- Added a CSS safety rule to hide stale `h4` slot headers on rendered TinyMCE
  article pages.
- Tuned plain paragraph splitting and page-full thresholds to reduce avoidable
  bottom gaps without knowingly clipping image/table/blockquote blocks.

Expected behaviour after deploying v3.4.4
-----------------------------------------

- Generated TinyMCE article pages should not show the external Article Library
  title at the top of every page.
- Whitespace-only optional page header/headline overrides should behave as blank.
- Opening and saving generated article pages in Issue Builder should not make
  page 2 revert to page 1 content.
- Existing v3.4.3 generated pages that have lost `articlePageHtml` may recover in
  preview if enough metadata remains, but the cleanest test is to place the
  article again after deploying v3.4.4.
- Page-bottom whitespace should improve for plain text, but large images/tables
  are still indivisible blocks and can still cause intentional gaps when the next
  block cannot fit safely.

Known limitations / next likely work
------------------------------------
The renderer is still client-side and measurement-based. It now preserves the
right generated page fragments, but final print fidelity should still move toward
server-side rendered page assets.

Suggested next development:

  v3.4.5 — Image Placement Controls / Renderer Preflight

Recommended focus:

1. Add explicit image placement controls:
   - inline
   - top of column
   - full-width title image
   - side-by-side figure pair
   - scale-to-remaining-column-height
2. Add a preflight render preview before committing article pages to the flatplan.
3. Show warnings for oversized images/tables/blocks in the UI, not just stored
   diagnostics.
4. Add a backend rendered page asset path:

     TinyMCE editable source
     -> controlled renderer
     -> immutable page preview image/PDF page
     -> Issue Builder places rendered page assets
     -> final PDF merge/export

Non-negotiable rules
--------------------
1. Magazine page count must stay a multiple of 4.
2. Never silently truncate article content.
3. Locked pages and locked panels are hard constraints.
4. Autosave should be the default behaviour.
5. Never write Python float values directly to DynamoDB.
6. User-facing names should be human-readable.
