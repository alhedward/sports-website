SUMMARY OF THIS PASS
====================
Tony reported two event calendar issues:

1. The magazine event calendar is usually a full page and should not be limited to the old “next 4 months” template idea. The editor needs to choose how many months appear in the event calendar.

2. In Advanced Layout, when selecting the page that already contains the event calendar, even when the page is not locked from regeneration, the calendar page appeared to refuse changes.

Changes made:

- Renamed the editor-facing event-calendar wording:
    Event calendar - next four months
  is now shown as:
    Event calendar - selected months

  The template ID remains `event_calendar_next_four_months` for compatibility with existing saved flatplans. Client-side template normalisation also overrides stale bootstrap display names so old server template metadata should still appear with selected-month wording.

- Backend and frontend calendar month limits increased from 12 to 24 months.

- Calendar editor controls now explain the intended workflow:
    choose months -> refresh calendar -> review/edit descriptions -> save page layout.

- Refresh button wording changed to:
    Refresh calendar using selected months

- Changing the months input now refreshes the calendar snapshot from club events and queues autosave.

- Calendar refresh preserves the editor’s existing short descriptions for events that remain in the refreshed date range.

- Added calendar slot alias handling so old pages using slot ID `calendar` and newer pages using slot ID `event_calendar` can both be edited using current template controls.

- Fixed likely save refusal root cause:
    `magazineSavePageSlotLayout()` previously awaited `magazineSyncCalendarShortDescriptions(layoutSlots)` before saving the page layout. If the event-record short-description sync failed, for example due to admin endpoint permissions or backend refusal, the page layout save was aborted.

  Now the calendar short-description sync is best-effort. The magazine page layout still saves even if the live event record update fails. A warning toast is shown only when the user does a manual save.

IMPORTANT CONTEXT
=================
Event calendar page editing still uses the Advanced Layout/Page Items machinery underneath, but the editor-facing workflow should continue moving toward the Issue Composer idea:

  The editor supplies content and makes editorial decisions.
  The composer arranges, refreshes, protects locks, and fits the issue.

Do not expose low-level slot concepts unnecessarily.
