Baseline package: v3.4.15 — Article Flow / Phantom Page Fix

Current focus:
- Magazine Article Library -> Issue Builder direct placement.
- TinyMCE source article must keep semantic formatting, including lists, spans, headings and captions.
- The renderer uses a native two-column CSS article sheet and creates clipped page windows for issue pages.
- Existing LROC footer style must be retained.

Confirmed from testing before this release:
- v3.4.13 double-click image caption editing works well.
- Captions render correctly in the Issue Builder preview.
- Font formatting is retained after the Article Library source was repaired.

What changed in v3.4.15:
- Kept the v3.4.13 caption behaviour.
- Changed image move controls from CSS margin nudges into real DOM/source-order moves:
  - Move selected image later in article flow.
  - Move selected image earlier in article flow.
- Added automatic whitespace mitigation at placement time by pulling following text blocks ahead of a movable image when that image would otherwise strand a large blank area at the bottom of a column.
- Added measured page-window content detection so a blank multicolumn tail does not create a phantom continuation page.
- Updated diagnostics to report measured page count and flow adjustments.

Known limitations / next likely work:
- The automatic whitespace pass only reorders whole following text blocks ahead of movable images. It deliberately does not split lists or rebuild columns.
- If there are still awkward holes, the next likely work should be a controlled placement UI for image policies: inline, after next paragraph, top of next column, shrink percent, or full-width feature image.
- Avoid returning to the explicit-column block packer from v3.4.7/v3.4.9; it damaged list formatting and worsened whitespace.
