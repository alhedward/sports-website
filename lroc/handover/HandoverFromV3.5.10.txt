What changed in v3.5.10:
  - Added editor-facing Issue Composer tab.
  - Magazine production is now framed as content-first composition:
      1. generate known recurring club pages,
      2. carry forward/place adverts,
      3. place edited articles,
      4. fill remaining gaps,
      5. final polish/review.
  - Old Issue Builder tab is now Advanced Layout.
  - Advanced Layout keeps the existing slot/page-item/page tools available for exceptions and fine tuning.
  - Issue Composer shows status cards:
      pages,
      generated pages,
      advert pages/panels,
      article pages,
      blank interiors,
      locked items.
  - Composer action buttons jump to relevant areas/pages instead of asking the editor to pick low-level methods.
  - Opening an issue now lands on Issue Composer instead of Issue Detail.
  - Article Library button wording changed to “Place article with composer”.
  - Legacy/manual placement wording was softened so it is clearly an exception path.
  - No backend placement semantics were changed in this pass.

Key design decision:
  - Tony clarified that the editor should provide content and editorial decisions.
  - The app should behave as the composer / Wizard of Oz:
      editor provides ingredients,
      system arranges the issue so it fits nicely,
      advanced layout machinery stays behind the curtain.

Current simplified workflow target:
  - Known site content pages:
      events,
      committee,
      roles,
      regular notices.
  - Adverts:
      deliberate placements,
      possible recurring/carry-forward behaviour later,
      lock panels that must survive article flow.
  - Articles:
      edited in TinyMCE Article Library,
      composer places into available issue space.
  - Imported/filler content:
      used after articles, adverts and generated content are settled.
  - Final polish/refit:
      later pass to reduce article-end blank space and balance typography across the finished issue.
