LROC handover from v3.4.11 — Article Renderer Page Window Fix
Timestamp: 2026-06-14 15:43:41 AEST

Baseline:
- v3.4.10 improved the visual article flow by returning to a native CSS multi-column sheet, but testing showed page 1 rendered again on continuation pages and the article expanded to four generated pages instead of the expected three.

Important lesson:
- The clipped-window approach is still the better direction than destructive block packing, but the horizontal page offset must be stored and preserved everywhere the Issue Builder reads/writes layout slots.
- Do not reintroduce the v3.4.7/v3.4.9 explicit-column/list-fragment packer unless there is a dedicated DOM-aware list handler.

v3.4.11 approach:
- Keep full TinyMCE HTML intact.
- Generate pages as windows across the native two-column sheet.
- Preserve mcePageOffsetXPx, stride, gap, and image cap values through preview/editor/save cycles.
- Use native multi-column scroll width as the page-count authority. The old block estimate is now diagnostics only, except manual page breaks still define a minimum page count.
- Add explicit CSS marker fallback for ul/ol inside generated article windows so bullets/numbers remain visible even when native markers are clipped.
