What changed in v3.5.9:
  - Bottom banner row 2 no longer uses fixed 50/50 cells when both cells contain text.
  - The site URL / left cell and Facebook URL / right cell now auto-balance automatically.
  - The balancing estimates each text group’s rendered width and chooses a cell split so:
      start of left text to LHS edge ~= end of right text to RHS edge
  - The automatic split is clamped to 30%–70% so it stays visually sane.
  - If only one lower-row cell has text, that text is centred across the full lower row.
  - If both lower-row cells are blank, the previous behaviour remains: top row vertically centres in the bottom bar.
  - No new editor control was added.
  - Front-cover live preview, revert stack and save behaviour remain unchanged.

Important front-cover model as of v3.5.9:
  - Top and bottom bars are full-width cover bars.
  - Both bars can be moved up/down.
  - Shared top/bottom banner opacity affects only the banner background/chrome, not the text.
  - Bottom bar layout is row 1 full-width centred, row 2 auto-balanced between two centred cells.
  - Bottom row-2 auto-balancing targets equal outside text margins, not equal cell widths.
  - Bottom top row has its own font size.
  - Bottom row-2 cells share one font size.
