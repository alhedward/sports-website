LROC handover from v3.5.0 — Region-Based Placement Foundation

Baseline:
- Built forward from the clean TinyMCE path after abandoning the Layout Lab/GrapesJS experiment.
- Article Library/TinyMCE remains the production composition path.
- Existing caption, advert-caption suppression, phantom-page trim, and deployer work are intended to remain intact.

Main change:
- The Issue Builder placement model is starting to move from rigid slot types to placed content regions.
- A page should be interpreted as occupied regions plus vacant regions.
- TinyMCE article continuations should eventually flow into suitable vacant regions in order, including half-page vacancies, rather than forcing all continuations to full pages.
- Content/region locks should eventually replace page-wide locks except for special pages such as covers or deliberately finalised pages.

Implemented in this package:
- Generated TinyMCE article page slots now preserve existing placed content as regions when calculating available article space.
- Stale generated slots for the same article are still removed during replacement.
- Article slots record placement region metadata such as `placementRegionKind`, `placementRegionBasis`, and `preservedRegionCount`.
- Shared helper namespace `window.mag350IssueRegions` exposes occupied/vacant region calculations for future planner/preflight work.

Next recommended work:
1. Add a visible placement preflight: show which page regions will be consumed before committing placement.
2. Implement true stream slicing across variable-sized regions.
3. Convert page-lock UI toward content/region locks.
4. Add A3 final printer/imposition export planning later; do not change the A4 authoring model yet.
