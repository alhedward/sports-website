SUMMARY OF THIS PASS
====================
Tony asked what to do first after agreeing the magazine should move toward
region/page-item based handling for mixed, bitty pages.

Decision:

  Start with the Issue Composer page-item foundation, not the full placement
  engine yet.

Changes made:

- Added a Composer page-item/region foundation view.

- Composer now has an issue page strip showing each page as fixed, blank, mixed,
  item-bearing and/or locked.

- Composer now lets the editor inspect a selected page without going straight
  into old slot-picking mode.

- The selected page shows:
    - item inventory cards,
    - page item kind labels,
    - item geometry summary,
    - lock state,
    - a simple A4 mini-map,
    - detected free regions,
    - a first preflight explanation.

- Page item kind labels currently include:
    - Advert,
    - Generated block,
    - Article flow,
    - Filler,
    - Image/photo,
    - Text/notice,
    - Locked panel,
    - Panel.

- Added Composer-side Lock item / Unlock item controls.
  These save panelLocked on the selected layout/content item via the existing
  /member/magazine/flatplan/page endpoint.

- Exposed window.mag3513Composer with helper functions:
    - pageItems(page),
    - vacantRegions(page),
    - pagePreflightHtml(page),
    - setItemLock(pageNumber, source, index, locked).

- Advanced Layout was deliberately left unchanged and remains the workshop path
  for manual exceptions.

IMPORTANT DESIGN INTENT
=======================
This version is a foundation pass only.

It does not yet auto-place articles or reflow pages. It gives Composer a clear,
editor-facing view of the physical page-item model so the next placement work can
be built safely.

The intended production rule remains:

  Fixed stuff first.
  Flowing article text second.
  Filler last.

For bitty mixed pages, the page is treated as a normal A4 canvas containing
multiple page items/regions rather than as a special exception.
