LROC Handover From v3.4.13
==========================

Current version: 3.4.13
Theme: Article Caption Control Fix
Timestamp basis: AEST / UTC+10

Context
-------
The v3.4.12 renderer got close: font formatting was retained and continuation page count returned to the expected range. Tony then reported that adding captions in the Article Library was not discoverable: left-clicking an image only showed TinyMCE resize handles and right-clicking showed a link widget. The previous caption guidance had effectively disappeared into the ether.

What changed in v3.4.13
-----------------------
- Added a visible Article Library button: "Caption selected image".
- Added double-click caption editing for images/figures in TinyMCE.
- Stopped using image/asset filenames as default captions for normal article images.
- Added filename-caption suppression for existing/imported captions such as image1.jpeg, image2.jpg, photo.png, etc.
- Empty captions stay visible as a placeholder inside the editor but are hidden in rendered issue pages.
- Kept the v3.4.12 continuation-window article renderer path.
- Kept the LROC footer style.

Testing priorities
------------------
1. In Article Library, click an inserted image and use "Caption selected image".
2. Double-click an image and confirm the caption prompt appears.
3. Confirm image filenames are no longer displayed as captions in Issue Builder render.
4. Confirm real manually entered captions are rendered below images.
5. Re-check article page count and continuation pages from a freshly placed article.

Known follow-up work
--------------------
- White-space tuning still needs a careful controlled strategy: likely image nudge/shrink controls rather than destructive block repacking.
- Issue Builder should eventually expose per-article image placement hints, but without damaging the source TinyMCE HTML.
- Deploy helper GitHub Actions log output is useful but still mostly post-step rather than true live browser-equivalent streaming.
